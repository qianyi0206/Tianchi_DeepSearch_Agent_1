# 可联网检索智能体方案（多线索历史/人物/机构谜题）

## 目标
构建一个可以联网检索、对齐多条线索并给出可解释答案的智能体，适配历史人物/机构/事件类谜题。

## 总体架构
1. 任务解析器
- 将题目拆成线索片段、时间约束、地域/领域约束
- 输出结构化约束：时间锚点、事件类型、地域、角色特征

2. 检索规划器
- 根据线索生成 3–6 条子查询，优先时间锚点
- 示例：
  - “French astronomer comet spectrum first observation year”
  - “sunspot photograph exhibition East Asia city astronomy exhibition year”
  - “southern European publisher founded hometown age 18 moved HQ to northern commercial center”

3. 检索与证据池
- 多源检索（百科、学术、博物馆/展览页面、企业史）
- 抽取网页片段并去噪
- 记录来源、时间、可信度

4. 时间线对齐器
- 把证据中的年份、地点、人物统一到时间线
- 寻找同年交集；若不唯一，继续检索缩小范围

5. 候选生成器
- 根据“出版业+南欧+创业者年龄+迁总部到北部商业中心”生成候选
- 逐个验证年份与迁移记录

6. 答案生成器
- 输出最终答案
- 可选：解释链或仅答案
- 标注不确定性与待验证点

## 关键能力设计
- 时间锚点优先：先锁年份再找人/公司
- 跨领域对齐：天文史 + 展览史 + 企业史
- 多来源验证：至少 2–3 个来源确认关键事实
- “十余年后”处理：验证迁移年份≈创立年份+10~15

## 执行流程（简化）
1. 解析线索 → 结构化约束
2. 生成子查询 → 搜索
3. 抽取时间锚点 → 年份候选
4. 依据年份找出版史线索
5. 验证创业者年龄、地点、迁总部时间
6. 生成答案 + 证据链

## 最小可用版本（MVP）
- 模块：解析器 → 检索 → 时间线对齐 → 答案生成
- 解析与年份抽取：用规则（正则/关键词）即可
- 搜索：先用简单搜索 API
- 证据池：结构化 JSON（标题、url、摘录、时间、可信度）

## 输出模式
- 答案模式：仅结论
- 解释模式：关键年份与证据
- 可追溯模式：完整证据链

## 可扩展点
- 引入重排序模型对证据做相关性排序
- 引入多轮检索：先锁年份，再补齐人物/机构
- 引入引用管理：输出时自动生成引用
